# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`zevm` is an Ethereum EVM implementation written in Zig (minimum version 0.14.1). It is a port/reimplementation of the EVM execution engine, currently at an early stage — core data structures are implemented but the full opcode interpreter is not yet complete.

## Build and test commands

```bash
# Build the executable
zig build

# Run all tests across all modules
zig build test

# Run tests for a specific module
zig build test:common
zig build test:vm
zig build test:core
zig build test:unit
zig build test:spec

# Filter tests by name within a module (e.g. only tests matching "jumpdest")
zig build test:vm -Dvm.filters=jumpdest

# Run the executable
zig build run:zevm
```

> `build.zig` is generated by `zbuild` from `zbuild.zon` — do not edit `build.zig` manually. Edit `zbuild.zon` instead when adding modules, executables, or tests.

## Module structure

The build defines four source modules and two test modules:

| Module | Root file | Purpose |
|---|---|---|
| `common` | `src/common/root.zig` | Primitive types: `Hash` (32 bytes), `Address` (20 bytes), hex encoding/decoding |
| `vm` | `src/vm/root.zig` | EVM execution primitives: `Stack`, `Memory`, `Contract`, `CodeBitmap`, `JumpDestCache`, `ScopeContext` |
| `core` | `src/root.zig` | Re-exports `common` and `vm`; used as the main library surface |
| `zevm` | `src/main.zig` | Executable entry point |
| `unit` (test) | `src/root.zig` | Alias of `core` used for unit test step |
| `spec` (test) | `test/spec/root.zig` | Placeholder for future Ethereum spec test vectors |

Module imports: `vm` and `zevm` import `common`; `core`/`unit` import `common`.

## Key types and ownership patterns

**`Stack`** (`src/vm/stack.zig`) — `u256` word stack, max 1024 entries. Uses `std.ArrayListUnmanaged`; callers pass the allocator explicitly to `push`/`dup`/`deinit`.

**`Memory`** (`src/vm/memory.zig`) — byte-addressed linear memory. Must call `resize` before `set`/`getPtr`. `set32` writes a `u256` word as big-endian.

**`Contract`** (`src/vm/contract.zig`) — holds bytecode, caller/address, gas, value, input. Borrows a `*JumpDestCache` (shared, EVM-lifetime). The `analysis` field is either borrowed from the cache (deployed code, identified by non-zero `code_hash`) or locally owned (initcode, `owns_analysis = true`). `deinit` frees local analysis only.

**`CodeBitmap`** (`src/vm/code_bitmap.zig`) — bit array marking PUSH-data bytes. A 0 bit = opcode space; a 1 bit = data. `codeSegment(pos)` returns `true` for opcode positions. Ownership depends on producer: `JumpDestCache` owns cache-stored bitmaps; callers own bitmaps from `codeBitmap()`/`CodeBitmap.fromCode()`.

**`JumpDestCache`** (`src/vm/jump_dest_cache.zig`) — `Hash → []u8` map of pre-analyzed bitmaps. `parseAndStore` is the main entry point. `load` returns a borrowed `CodeBitmap`; its backing storage belongs to the cache.

**`ScopeContext`** (`src/vm/interpreter.zig`) — groups `*Memory`, `*Stack`, and `*Contract` for a single call frame. All fields are pointers; callers manage lifetimes.

**`common.Hash` / `common.Address`** (`src/common/types.zig`) — fixed-byte arrays with hex encoding. `Address.hex()` returns EIP-55 checksum encoding. Both accept hex strings with or without `0x` prefix and right-align shorter inputs.

## Tracing

`src/vm/tracing/gas_change_reason.zig` defines the `GasChangeReason` enum used by `Contract.useGas` and `Contract.refundGas`. The logger parameter in those functions is currently unused (`anytype`, accepts `null`).

## Conventions

- Tests live inline in the same file as the code they test, after the `pub const` declarations.
- `std.debug.assert` is used for preconditions (not `error` returns) — violations are safety-checked undefined behavior in Debug mode.
- Module roots (`root.zig`) re-export public symbols; import the module name rather than individual files from outside the module.
